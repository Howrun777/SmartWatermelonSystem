# 瓜田监测可视化平台 (Web前端) - 开发者文档

## 1. 系统概述
本系统是“智能西瓜无损糖度检测系统”的应用层（Web端）。基于纯原生技术栈（Vanilla HTML/CSS/JS + ECharts），旨在提供一个现代化、防断网、高交互的 IoT 监控大屏（Dashboard）。

**核心能力：**
*   **安全防御**：基于 Cookie (Session) 与后端 C++ 进行鉴权，无权限者触发 HTTP 302 重定向拦截。
*   **实时驱动**：具备 6 秒静默数据轮询机制与强制手动刷新功能，保持数据最新状态。
*   **智能交互**：支持 60 秒无操作自动切换瓜田巡航；支持对多维度图表进行动态时间尺度（5分钟/2小时/1天/1周）的平滑切换与真实时间轴映射。
*   **UI 风格**：采用现代化拟态玻璃（Glassmorphism）、响应式 Grid 布局、定制化 ECharts 坐标系渲染（红/黑/黄状态线刻画）。

## 2. 技术栈与目录结构

**技术选型：**
*   **视图层**：HTML5 + 原生 CSS3 (CSS Variables, Flexbox, Grid)
*   **逻辑层**：原生 JavaScript (ES6+, Async/Await, Fetch API)
*   **可视化**：ECharts v5.5.0 (支持离线/断网加载)
*   **部署方式**：由 C++ 后端 (`cpp-httplib`) 挂载静态目录直接托管。

## 3. 核心功能与模块解析

### 3.1 登录与权限模块 (`login.html` & `api.js`)
*   **业务逻辑**：收集用户输入的账密，调用 `/api/admin/login`。
*   **鉴权机制**：登录成功后，后端会在响应头中写入 `Set-Cookie: session_id=xxx`。前端利用该机制，后续所有 Fetch 请求会自动携带 Cookie。
*   **安全拦截**：`dashboard.html` 并非直接访问静态文件，而是由后端 C++ 路由提供。如果 Cookie 失效，后端返回 302 强行重定向至 `login.html`。

### 3.2 大屏视图结构 (`dashboard.html`)
采用 100vh 全屏 `Flexbox` 布局，分为顶部导航栏和下方左右双面板。
*   **顶部导航**：包含瓜田切换（◀/▶）、当前系统时间、最后同步时间、强制刷新按钮。
*   **左侧面板 (flex: 4)**：
    *   顶部：西瓜状态阵列（`CSS Grid`），展示每个西瓜的设备ID、糖度值、动态计算的成熟度状态（红/黄标）。
    *   底部：单果历史糖度折线图（需点击阵列卡片触发联动）。
*   **右侧面板 (flex: 5)**：
    *   三个纵向排列的环境历史折线图：温度(℃)、湿度(%)、光照(Lux)。
*   **时间控制组件 (`.scale-control`)**：每个图表右上角独立挂载控制按钮，允许切换 4 种时间尺度。

### 3.3 图表渲染引擎 (`chart.js`)
这是前端视觉的核心，摒弃了 ECharts 的默认自适应配置，采用**“降维打击”**的严格控制模式。

*   **X 轴底层逻辑 (核心)**：强制使用 `type: 'value'`（纯数值轴）而非 `'time'` 轴。通过传入精确的毫秒时间戳作为边界（`min`/`max`）和刻度间隔（`interval`），强制 ECharts 绘制绝对均匀的 12 个刻度。通过 `formatter` 动态将数字戳还原为可读时间（时:分 或 月.日）。
*   **Y 轴色彩映射逻辑**：
    *   抛弃常规的阈值线，直接通过传入颜色数组给 `yAxis.splitLine.lineStyle.color`。
    *   实现例如温度在 `[-10,0]`和`[40,60]`为红色告警线，`[10,30]`为黑色正常线的严格工业级 UI 还原。
    *   单独为糖度表注入黄色的 `markLine` 作为成熟阈值基准线。
*   **Tooltip 接管**：因 X 轴被降维为数值轴，自带的提示框会显示庞大的毫秒数，故重写 `tooltip.formatter`，将悬浮点的时间戳翻译回标准的 `YYYY年MM月DD日 HH:mm:ss`。

### 3.4 业务主控中心 (`auto_switch.js`)
统筹全局状态管理、定时器轮询与数据网络请求。

*   **全局状态**：维护 `currentFieldId` (当前瓜田), `currentSelectedWmId` (当前选中西瓜), 以及 4 个图表的独立 `scales` 状态 (0~3 级)。
*   **双定时器机制**：
    *   `idleTimer` (宏观防打扰)：60 秒无任何鼠标/键盘交互时，自动调用 `switchField(1)` 切换到下一个瓜田，实现大屏无人值守巡航。任何交互操作会调用 `resetIdleTimer()` 重置计时。
    *   `dataRefreshTimer` (微观静默同步)：永久每 6 秒运行一次，调用 `reloadAllChartsData()` 静默拉取后端最新数据重绘图表，不触发闪烁动画。
*   **数据映射逻辑 (`reloadAllChartsData`)**：
    *   通过 `fetch` 获取后端一次性下发的 84 天全量数据。
    *   调用 `getTimeRange(scaleLevel)` 根据当前选中的刻度级别（5min / 2h / 1d / 1week），精确计算出过去的时间戳起点 `min`、当前时间点 `max`，以及固定的间隔跨度 `interval`。
    *   将后端的 JSON 对象数组提取为 ECharts 要求的 `[[时间戳, 数值], [时间戳, 数值]]` 散点格式，直接投喂给 `chart.js`。即使数据存在断档断联，散点也会依据时间戳精确落在正确的坐标轴物理位置上。

## 4. 前后端通信 API 契约

前端 `auto_switch.js` 依赖以下由 C++ 提供的 RESTful API：

1.  **获取瓜田列表**：`GET /api/admin/field/list`
    *   响应提取：`json.data.list.map(item => item.field_id)`
2.  **获取瓜田下所有西瓜的最新实时数据**：`GET /api/admin/watermelon/list?field_id={id}`
    *   用途：渲染左侧 `renderWatermelonGrid` 阵列卡片。
3.  **获取单瓜田 84 天环境数据**：`GET /api/admin/field/environment?field_id={id}`
    *   响应提取：遍历组装为 `[timestamp*1000, temperature/humidity/light]` 的散点。
4.  **获取单西瓜 84 天糖度数据**：`GET /api/admin/watermelon/history?device_id={id}`
    *   前提：左侧西瓜阵列必须存在至少一个选中的西瓜（`currentSelectedWmId`）。

## 5. 开发调试指引 (给后续 AI 的提示)

如果你（AI）需要修改该 Web 端代码，请注意以下边界条件：
1.  **禁止重构技术栈**：保持纯 Vanilla JS 架构，绝对不要引入 Vue、React、Webpack 或 Babel，本项目要求直接在浏览器中运行源码。
2.  **图表 X 轴不可更改**：禁止将 `chart.js` 中的 `xAxis.type` 改回 `'time'` 轴，否则会导致 12 刻度均匀对齐规则崩溃。
3.  **开发环境跨域限制**：本地直接双击 HTML 文件会导致 Cookie 写入失败以及 CORS 报错。调试必须依赖后端的 C++ HttpServer 托管（挂载点配置），通过 `http://47.107.41.102:8080/index.html` 进行访问。
4.  **样式修改原则**：尽量使用 `style.css` 顶部的 CSS Variables (`--primary-color` 等) 调色，避免在 JS 中硬编码色值。
