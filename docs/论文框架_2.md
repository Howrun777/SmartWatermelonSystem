Design and Implementation of Intelligent Non-Destructive Watermelon Sugar Content Detection System Based on ESP32
（三号黑体，居中）

Abstract
（三号Times New Roman，居中，空1行）

Aiming at the actual demand of field quality control in China's watermelon industry and on-site detection at the consumer end, as well as the industry pain points of complex operation of traditional destructive detection methods and the high cost of large-scale spectral equipment, this paper designs and implements a portable intelligent non-destructive watermelon sugar content detection system based on ESP32 main control chip. The system takes AS7341 near-infrared spectrum sensor as the detection core, integrates SHT31 environmental parameter acquisition unit and DS3231 high-precision real-time clock unit, and adopts LVGL graphics library to realize touch-based human-computer interaction interface. The system designs a dual business architecture of consumer test mode and production industrial mode, supporting core functions such as LittleFS local persistent storage, random off-peak automatic reissue after WiFi recovery, dual-mechanism time synchronization, and sensor abnormal decoupling. A C++ back-end data service system is developed to realize device identity verification, dual-index calculation of sugar content and maturity, and Session-based session management. A watermelon field monitoring visualization Web platform is built based on native HTML/CSS/JS, supporting watermelon card array display, 30-minute offline alarm, 60-second automatic field cruising switch, and ECharts multi-scale data visualization. The test results show that the system has a sugar content detection error ≤ ±0.5°Brix, supports up to 1000 offline data storage, and can stably run in multiple scenarios such as field and supermarket, meeting the needs of on-site non-destructive detection at both planting and consumer ends, with good practical value.

##### 2.1.1 功能性需求分析

（3级标题，小四号黑体，不加粗）

- 1:1匹配实际项目，拆解为9项核心功能：
  1. **时间同步需求**：DS3231硬件时钟优先，WiFi连接后自动向服务器校时，每次数据上传后用响应中的server_time再次校准，DS3231电池耗尽时降级使用ESP32内部时钟
  2. **数据采集需求**：AS7341采集10通道光谱数据（ch415/ch445/ch480/ch515/ch555/ch595/ch640/ch680/Clear/NIR），SHT31采集温湿度（可选，缺失则填99.0），BH1750采集光照（可选，缺失则填99）
  3. **传感器解耦需求**：环境传感器缺失时不影响光谱采集核心功能，数据标记为99而非阻断测量流程
  4. **双模式业务需求**：测试模式（内存存储≤20条、不上传、不采集环境数据）；工业模式（LittleFS持久化、每5分钟自动检测、HTTP上传、离线存储+断网补发）
  5. **本地存储需求**：LittleFS JSON Lines格式存储，存储空间>80%时自动清理50%最旧记录（优先删除已上传数据），最多支持1000条
  6. **网络通信需求**：WiFi断线每30秒自动重连，连网后随机0~10秒延迟补发未上传记录（新数据优先上传）
  7. **后端接口需求**：设备校时GET接口、数据上传POST接口（支持spectrum_json+环境数据+可选时间戳）、管理员Session登录接口
  8. **前端可视化需求**：西瓜卡片实时展示糖度/成熟度/在线状态，ECharts图表多尺度时间导航，60秒无操作自动切换瓜田，6秒静默刷新，30分钟掉线卡片变红告警
  9. **安全性需求**：设备端device_id+token身份校验，后端Argon2id密码哈希，Session Cookie 1小时过期

##### 2.1.2 非功能性需求分析

（3级标题，小四号黑体，不加粗）

- 明确系统量化性能指标：
  1. 检测精度：糖度检测绝对误差≤±0.5°Brix
  2. 系统响应：开机启动时间≤3s，单次光谱采集响应时间≤2s
  3. 存储能力：支持最大1000条离线数据存储，80%容量自动触发50%清理
  4. 网络重连：WiFi断开后每30秒尝试重连，不阻塞UI主循环
  5. 并发能力：后端支持≥100台设备同时接入，设备端随机0~10秒错峰防止补发雪崩
  6. 稳定性：连续72小时无故障运行
  7. 工作环境：设备端温度范围0℃~~45℃，适配田间（0℃~~40℃）、商超场景

#### 2.2 系统总体架构设计

（2级标题，四号黑体，加粗）

- 采用三层物联网架构设计，绘制系统总体架构框图，说明各层职责：
  1. **终端感知层（FieldAcquisitionTerminal）**：ESP32主控 + AS7341光谱传感器 + SHT31/BH1750环境传感器 + DS3231 RTC + 2.8寸TFT触摸显示屏 + 3W LED光源，负责数据采集、本地糖度MLR计算、LVGL人机交互、LittleFS离线存储、HTTP网络上传
  2. **数据服务层（FieldDataProcessingServer）**：C++后端服务（cpp-httplib） + MySQL数据库（sws_db），负责设备身份校验、糖度与成熟度双指标计算、Session会话管理、RESTful API提供、静态前端资源托管
  3. **应用展示层（FieldMonitoringPlatform）**：原生HTML/CSS/JS Web前端（由后端直接托管）+ ECharts 5.5可视化，负责管理员认证登录、西瓜卡片阵列展示、糖度/环境历史图表、掉线告警、自动巡航

#### 2.3 相关技术基础

（2级标题，四号黑体，加粗）

##### 2.3.1 近红外光谱糖度检测原理

（3级标题，小四号黑体，不加粗）
