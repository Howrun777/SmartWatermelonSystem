## 项目概述

### 系统简介
**智能西瓜采集系统** 是一款田间采集终端设备，用于实时采集西瓜的光谱、环境数据，计算糖度等重要指标，支持离线测试和在线工业模式两种运行方式。

### 核心特性
- ✅ **多传感器集成**：光谱（AS7341）、温湿度（SHT31）、光照（BH1750）
- ✅ **宽谱LED控制**：精准的光源管理
- ✅ **实时UI显示**：基于LVGL的ILI9341触摸屏界面
- ✅ **WiFi上传**：支持实时数据上传到云端
- ✅ **离线模式**：测试模式支持本地数据累积
- ✅ **错峰补发**：WiFi断连后自动重连并补发数据
- ✅ **时钟同步**：DS3231实时时钟与服务器校时

### 硬件平台
| 组件 | 型号 |
|------|------|
| 微控制器 | ESP32 |
| 显示屏 | ILI9341（240×320） |
| 触摸屏 | XPT2046 |
| 光谱传感器 | AS7341 (I2C) |
| 温湿度传感器 | SHT31 (I2C) |
| 光照传感器 | BH1750 (I2C) |
| 实时时钟 | DS3231 (I2C) |
| LED驱动 | GPIO 26 |

## 硬件架构

### 引脚分配

#### I2C总线（传感器）
```
SDA: GPIO 27
SCL: GPIO 22
设备地址:
  - AS7341:  0x39
  - SHT31:   0x44
  - BH1750:  0x23
  - DS3231:  0x68
```

#### SPI总线（显示屏）
```
TFT_eSPI配置 (已在platformio.ini中定义):
  - MISO: GPIO 12
  - MOSI: GPIO 13
  - SCLK: GPIO 14
  - CS:   GPIO 15
  - DC:   GPIO 2
  - BL:   GPIO 21 (背光)
```

#### 独立SPI总线（触摸屏）
```
触摸屏使用独立SPI总线 (HSPI):
  - MOSI: GPIO 32
  - MISO: GPIO 39
  - CLK:  GPIO 25
  - CS:   GPIO 33
  - IRQ:  GPIO 36
```

#### 其他GPIO
```
LED_PIN: GPIO 26 (宽谱LED控制)
```

### 电源管理
- 主供电：5V USB供电
- 内部稳压：3.3V（ESP32内部）
- 传感器供电：3.3V

### 核心模块说明

#### 1. **SensorManager** - 传感器管理
**位置**: [src/sensor_driver/SensorManager.h](src/sensor_driver/SensorManager.h) & [.cpp](src/sensor_driver/SensorManager.cpp)

**功能**:
- 初始化所有I2C传感器（AS7341、SHT31、BH1750）
- 读取环境数据（温度、湿度、光照）
- 读取光谱数据（10个通道）
- 控制宽谱LED开关

**关键接口**:
```cpp
void begin();                                    // 初始化
void readEnvironment(float &t, float &h, int &l); // 读环境
bool readSpectrum(JsonObject& doc);              // 读光谱
void controlLight(bool state);                   // LED控制
```

#### 2. **UIManager** - UI显示管理
**位置**: [src/display/lvgl_ui.h](src/display/lvgl_ui.h) & [.cpp](src/display/lvgl_ui.cpp)

**功能**:
- 基于LVGL框架的触摸UI
- 两种模式界面（测试模式 / 工业模式）
- 实时数据显示（糖度、环境、光谱）
- 数据历史记录

**工作流程**:
```
启动 → 启动屏 → 模式选择 → 工业/测试界面 → 实时更新
```

#### 3. **SugarCalc** - 糖度计算
**位置**: [src/algorithm/sugar_calc.h](src/algorithm/sugar_calc.h)

**算法**: 多元线性回归 (MLR)
```cpp
Brix = a×CH415 + b×CH445 + c×CH480 + k
其中:
  a = 0.0012, b = 0.0005, c = 0.0021, k = 5.2
  取值范围: [0, 20]
```

#### 4. **HttpClient** - 网络通讯
**位置**: [src/network/HttpClient.h](src/network/HttpClient.h) & [.cpp](src/network/HttpClient.cpp)

**功能**:
- WiFi连接管理（非阻塞，最多10秒超时）
- 数据上传到云端
- 服务器时间同步
- 自动重连（30秒间隔）

**关键接口**:
```cpp
static bool connectWiFi();                                    // 连接WiFi
static void maintainWiFi();                                  // 维护连接
static uint32_t fetchServerTime();                           // 获取服务器时间
static bool uploadData(const JsonDocument& payload, 
                       String& outMsg, 
                       uint32_t* outServerTime);             // 上传数据
```

## 模块详解

### 1. 传感器驱动层

#### AS7341 光谱传感器
**文件**: [src/sensor_driver/AS7341.h/cpp](src/sensor_driver/AS7341.h)

**通道说明**:
| 通道 | 波长 | 用途 |
|------|------|------|
| CH415 | 415nm | 蓝色 |
| CH445 | 445nm | 蓝色 |
| CH480 | 480nm | 蓝色 |
| CH515 | 515nm | 绿色 |
| CH555 | 555nm | 绿色 |
| CH595 | 595nm | 黄色 |
| CH640 | 640nm | 红色 |
| CH680 | 680nm | 红色 |
| CLR | - | 清晰度 |
| NIR | - | 红外 |

#### SHT31 温湿度传感器
**精度**:
- 温度：±0.3°C
- 湿度：±2% RH

#### BH1750 光照传感器
**范围**: 0 - 65535 lux

#### DS3231 实时时钟
**精度**: ±2 ppm
**用途**: 离线时钟维护

### 2. UI模块

#### 系统模式
```cpp
enum SystemMode {
    MODE_WAITING = 0,      // 待命 - 显示启动屏和模式选择
    MODE_TEST = 1,         // 测试模式 - 本地累积20条数据
    MODE_INDUSTRIAL = 2    // 工业模式 - 每5分钟自动测量上传
};
```

#### 测试模式
- 手动触发测量
- 累积最多20条数据
- 显示平均糖度
- 数据仅存于内存（不持久化）
- 可清空数据后重新测试

#### 工业模式
- 每5分钟自动测量一次
- 自动上传到服务器
- 显示历史记录
- WiFi断连时本地累积，恢复后补发

### 3. 网络通讯

#### 数据上传格式
```json
{
  "device_id": "1001-01-01",
  "device_token": "device-token-001",
  "watermelon_variety": "Kirin Watermelon",
  "timestamp": 1681234567,
  "temperature": 25.5,
  "humidity": 65.3,
  "light_intensity": 12000,
  "spectrum": {
    "ch415": 100,
    "ch445": 120,
    "ch480": 130,
    "ch515": 140,
    "ch555": 150,
    "ch595": 160,
    "ch640": 170,
    "ch680": 180,
    "clr": 500,
    "nir": 400
  },
  "brix": 13.5
}
```

#### WiFi管理机制
- **连接**: 非阻塞，最多等10秒
- **重连**: 每30秒尝试一次
- **时间同步**: 每次成功上传时同步服务器时间

### 4. 糖度计算

#### 离线算法
设备端使用本地MLR算法计算糖度，与服务器算法保持一致。

#### 优化
- 取值范围限制在 [0, 20] Brix
- 防止数据异常

## API参考

### SensorManager

#### `void begin()`
初始化所有传感器，检测设备连接状态。

```cpp
SensorManager sensorMgr;
sensorMgr.begin();  // 初始化时调用
```

#### `void readEnvironment(float &temp, float &hum, int &light)`
读取环境数据（温度、湿度、光照）。

```cpp
float t, h;
int l;
sensorMgr.readEnvironment(t, h, l);
// t: 温度 (°C)
// h: 湿度 (%)
// l: 光照 (lux)
```

#### `bool readSpectrum(JsonObject& doc)`
读取光谱数据，返回10个通道的数值。

```cpp
JsonDocument doc;
if (sensorMgr.readSpectrum(doc.as<JsonObject>())) {
    int ch415 = doc["ch415"];
    // ...
}
```

#### `void controlLight(bool state)`
控制宽谱LED。

```cpp
sensorMgr.controlLight(true);   // 开启LED
sensorMgr.controlLight(false);  // 关闭LED
```

### UIManager

#### `void init(MeasureCallback, ModeSelectCallback, ClearTestCallback, ReturnHomeCallback)`
初始化UI管理器。

```cpp
UIManager ui;
ui.init(
    onMeasureBtnClicked,
    onModeSelected,
    onClearTestClicked,
    onReturnHomeClicked
);
```

#### `void showBootScreen()`
显示启动屏幕。

#### `void showModeSelection()`
显示模式选择界面（测试/工业）。

#### `void updateEnv(float t, float h, int l)`
更新环境数据显示。

#### `void updateSpectrum(...)`
更新光谱数据显示（10个通道）。

#### `void updateSugar(float brix)`
更新糖度显示。

#### `void addIndustrialHistory(const char* timeStr, float brix, float temp, float hum, int light, bool uploaded)`
添加工业模式历史记录。

### HttpClient

#### `static bool connectWiFi()`
连接WiFi（非阻塞）。返回连接成功与否。

```cpp
if (NetworkManager::connectWiFi()) {
    Serial.println("WiFi已连接");
}
```

#### `static uint32_t fetchServerTime()`
从服务器获取当前时间戳。返回0表示失败。

```cpp
uint32_t serverTime = NetworkManager::fetchServerTime();
if (serverTime > 0) {
    // 校时
}
```

#### `static bool uploadData(const JsonDocument& payload, String& outMsg, uint32_t* outServerTime)`
上传数据到服务器。

```cpp
JsonDocument payload;
payload["device_id"] = "1001-01-01";
payload["brix"] = 13.5;

String msg;
uint32_t serverTime = 0;
if (NetworkManager::uploadData(payload, msg, &serverTime)) {
    Serial.println("上传成功");
}
```

### SugarCalc

#### `static float calculate(const JsonObject& spectrum)`
计算糖度。

```cpp
float brix = SugarCalc::calculate(spectrum);
// brix 范围: [0, 20]
```

## 工作流程

### 启动流程
```
setup() 
  ↓
初始化串口、硬件SPI、触摸SPI、显示屏
  ↓
初始化SensorManager（传感器）
  ↓
初始化UIManager（LVGL UI）
  ↓
尝试连接WiFi（非阻塞，最多10秒）
  ↓
从服务器同步时间（可选）
  ↓
显示启动屏和模式选择
```

### 测试模式工作流
```
用户选择测试模式
  ↓
显示测试UI
  ↓
用户点击"测量"按钮
  ↓
控制LED开启 → 等待500ms → 读取传感器
  ↓
计算糖度，本地累积（最多20条）
  ↓
更新UI显示 (糖度、平均值、数据表)
  ↓
用户可继续测量或清空数据
```

### 工业模式工作流
```
用户选择工业模式
  ↓
显示工业UI，启动倒计时（5分钟）
  ↓
倒计时完毕，自动测量
  ↓
读取传感器 → 计算糖度
  ↓
准备JSON数据包
  ↓
尝试上传到服务器
  ↓
  ├─ 成功 → 标记已上传，开始新倒计时
  │
  └─ 失败 → 数据本地累积，30秒后重试WiFi连接
      （恢复连接后补发所有待发送数据）
  ↓
如此循环...
```

### 错峰补发流程
```
WiFi断开
  ↓
数据本地累积
  ↓
系统每30秒尝试重连WiFi
  ↓
WiFi恢复
  ↓
补发所有待发送数据
  ↓
清空本地缓冲
```

## 常见问题

### Q1: 如何修改自动测量间隔？
编辑 [config.h](config.h)：
```cpp
#define AUTO_UPLOAD_INTERVAL 300000  // 改为需要的毫秒数
// 例如: 60000 为 1 分钟
```

### Q2: 如何自定义糖度计算公式？
编辑 [src/algorithm/sugar_calc.h](src/algorithm/sugar_calc.h) 中的 `calculate()` 方法：
```cpp
float brix = a * ch415 + b * ch445 + c * ch480 + k;  // 修改此行
```

### Q3: 传感器初始化失败怎么办？
检查：
1. I2C引脚连接是否正确（GPIO 27 为SDA，GPIO 22 为SCL）
2. 传感器地址是否正确
3. 电源供电是否稳定
4. 查看串口输出的日志信息

### Q4: WiFi无法连接怎么办？
1. 检查 [config.h](config.h) 中的WiFi SSID和密码
2. 确保设备在WiFi信号范围内
3. 检查防火墙设置
4. 查看目标服务器是否正常运行

### Q5: 数据上传失败，本地数据会丢失吗？
不会。系统采用错峰补发机制，数据会本地累积，待WiFi恢复后自动补发。

## 故障排查

### 问题：传感器读数异常
**可能原因**:
- I2C总线干扰
- 传感器故障
- 电源不稳定

**解决方案**:
1. 检查I2C连接
2. 尝试重启设备
3. 查看串口日志确认初始化状态

### 问题：UI显示卡顿
**可能原因**:
- LVGL缓冲区过小
- SPI频率设置过低
- CPU占用过高

**解决方案**:
- 在 [platformio.ini](platformio.ini) 中增加SPI频率或LVGL缓冲
- 检查是否有阻塞操作

### 问题：数据上传超时
**可能原因**:
- 网络连接不稳定
- 服务器响应缓慢
- 数据包过大

**解决方案**:
1. 检查WiFi信号强度
2. 尝试减小数据包大小
3. 增加上传超时时间

### 问题：时间不同步
**可能原因**:
- 服务器时间同步失败
- DS3231电池耗尽

**解决方案**:
1. 检查服务器是否可达
2. 检查DS3231电池电量
3. 手动校时或依赖服务器校时
